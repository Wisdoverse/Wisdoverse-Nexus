# 安全审计报告（2026-02-25）

## 1. 审计范围

- 代码与配置：
  - `crates/nexis-gateway/src/main.rs`
  - `.env.example`
  - `.github/workflows/security.yml`
  - `.gitignore`
  - `docs/security/*`
- 审计目标：
  - 安全头（CORS/CSP/HSTS/HTTPS）配置
  - 环境变量与密钥使用方式
  - 仓库内敏感信息泄露风险

## 2. 已实施安全措施

### 2.1 仓库与供应链

- 已启用 secrets 扫描：`gitleaks`（CI）。
- 已启用依赖漏洞审计：`cargo audit`（CI）。
- 已启用静态分析：CodeQL（CI）。
- `.gitignore` 已覆盖 `.env`、证书、私钥等高风险文件模式。

### 2.2 运行时与网关入口

- 网关新增可配置安全头：
  - `Content-Security-Policy`
  - `Strict-Transport-Security`
  - `X-Content-Type-Options`
  - `X-Frame-Options`
  - `Referrer-Policy`
  - `Permissions-Policy`
- CORS 从“全放开”改为“可配置白名单”：
  - `NEXIS_CORS_ALLOW_ORIGINS`
  - `NEXIS_CORS_ALLOW_CREDENTIALS`
- 支持 HTTPS 强制跳转（默认关闭，生产应开启）：
  - `NEXIS_HTTPS_REDIRECT_ENABLED`
- HSTS 与 CSP 策略可配置：
  - `NEXIS_HSTS_ENABLED`
  - `NEXIS_CSP_POLICY`

### 2.3 配置与密钥治理

- 所有 provider/API 密钥通过环境变量注入（未发现业务代码硬编码真实密钥）。
- `.env.example` 提供安全占位值与轮换周期参数（`NEXIS_SECRET_ROTATION_DAYS` 等）。
- 已补充密钥管理指南：`docs/security/key-management.md`。

## 3. 配置检查结果

### 3.1 敏感信息泄露扫描（本次）

- 检查方式：基于常见 key/token/私钥模式的仓库扫描。
- 结果：未发现真实生产密钥入库。
- 发现项：仅存在测试 token 与文档占位示例（非真实凭证）。

### 3.2 环境变量使用

- 网关、Runtime、MCP Provider 关键密钥均走 `process env` 读取。
- 未发现将密钥写入日志或返回响应体的直接逻辑。
- 新增安全相关环境变量已纳入 `.env.example`。

## 4. 潜在风险与改进建议

1. HTTPS 强制默认未开启（`NEXIS_HTTPS_REDIRECT_ENABLED=false`）。
2. WebSocket 当前仍存在 query token 使用场景，建议迁移到短时 ticket 或握手头。
3. 当前 HSTS 启用依赖环境变量，生产环境需纳入发布前强校验。
4. 缺少自动化容器镜像扫描与 SBOM 产物归档（仍是合规差距项）。

## 5. 结论

- 本轮完成后，安全合规关键缺口（安全头配置、配置检查、密钥管理文档化）已闭环。
- 建议将“生产必须开启 HTTPS Redirect + HSTS”加入发布 Gate，防止配置回退。
