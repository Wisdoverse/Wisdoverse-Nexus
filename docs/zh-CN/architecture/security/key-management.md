# 密钥管理指南

## 1. 适用范围

- 适用于 Nexis 的 API Key、JWT/HMAC 秘钥、Webhook 签名密钥、加密主密钥。
- 适用环境：Baseline 与 Enterprise。

## 2. 基本原则

1. 密钥只允许通过环境变量或 KMS/Vault 注入。
2. 禁止明文写入仓库、镜像层、日志与监控标签。
3. 最小权限访问：仅运行时进程与必要维护角色可读。
4. 所有轮换必须有工单与审计记录。

## 3. 密钥清单（示例）

| 变量 | 用途 | 建议轮换周期 |
|------|------|--------------|
| `OPENAI_API_KEY` | LLM Provider 调用 | 90 天 |
| `ANTHROPIC_API_KEY` | LLM Provider 调用 | 90 天 |
| `GEMINI_API_KEY` | LLM Provider 调用 | 90 天 |
| `NEXIS_API_HMAC_SECRET` | API 请求签名校验 | 90 天 |
| `NEXIS_WEBHOOK_SIGNING_KEY` | Webhook 验签 | 90 天 |
| `NEXIS_ENCRYPTION_KEY` | 应用层加密 | 180 天（或按合规要求） |
| `NEXIS_SSO_CLIENT_SECRET` | 企业 SSO 客户端密钥 | 90 天 |
| `NEXIS_KMS_KEY_ID` | KMS 密钥引用 | 依据 KMS 策略 |

## 4. 密钥轮换流程（SOP）

### 4.1 准备

1. 创建变更工单，标记影响系统与回滚窗口。
2. 在 KMS/Vault 生成新版本密钥，记录版本号。
3. 预先在 staging 注入新密钥并跑回归测试。

### 4.2 执行

1. 生产环境以灰度方式注入新密钥（先低风险实例）。
2. 观察 10-30 分钟：认证失败率、5xx、上游调用错误率。
3. 扩大到全量实例。
4. 旧密钥进入“禁用待删除”状态（建议 24 小时观察窗）。

### 4.3 验证

1. 验证关键链路可用：登录、鉴权 API、Webhook、Provider 调用。
2. 审计日志确认无旧密钥调用。
3. 完成工单，附上变更时间、验证结果、责任人。

### 4.4 回滚

1. 若异常，立即回退到上一版本密钥。
2. 记录异常症状与回滚时间线。
3. 提交复盘并更新下次轮换前置检查项。

## 5. 检查清单

- [ ] 代码仓库无真实密钥（secret scan 通过）
- [ ] `.env.example` 仅保留占位值
- [ ] 所有业务密钥都有明确 owner 与轮换周期
- [ ] 轮换执行有工单编号与审计证据
- [ ] 旧密钥已失效并从运行环境移除
