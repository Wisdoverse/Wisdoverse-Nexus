## 🚨 P0 问题（立即修复）

### 1. JWT 硬编码 ⚠️ 极高风险

**问题**: 生产环境可能使用默认密钥 "dev_only_secret_change_in_production"

**影响**: 任意用户可伪造 token，完全绕过认证

**修复时间**: 1-2 天

**修复方案**:
```rust
// 移除默认值，强制要求环境变量
let secret = std::env::var("JWT_SECRET")
    .expect("JWT_SECRET must be set");
```

---

### 2. CRDT 空壳 ⚠️ 核心功能缺失

**问题**: 仅内存实现，无持久化、无冲突解决算法

**影响**: 协作功能完全不可用

**修复时间**:
- 自研: 8-12 周
- 集成 yrs: 2-3 周 ✅ **强烈推荐**

**建议**: 集成 yrs（Yjs 的 Rust 实现），节省 6-9 周

---

### 3. SFU 空壳 ⚠️ 核心功能缺失

**问题**: 无 WebRTC 实现，仅内存状态管理

**影响**: 视频会议功能完全不可用

**修复时间**:
- 自研: 10-14 周
- 集成 LiveKit: 2-3 周 ✅ **强烈推荐**

**建议**: 集成 LiveKit，节省 8-11 周

---

### 4. 有状态网关 ⚠️ 无法扩展

**问题**: WebSocket 连接状态存储在内存，无法水平扩展

**影响**: 单点故障，无法支撑 100+ 并发

**修复时间**: 6-8 周

**修复方案**: 状态迁移到 Redis 或 PostgreSQL

---

### 5. 无 RLS（Row Level Security）⚠️ 数据泄露

**问题**: 数据库查询无租户隔离

**影响**: 用户可访问其他租户数据

**修复时间**: 2-3 周

**修复方案**:
```sql
ALTER TABLE rooms ENABLE ROW LEVEL SECURITY;
CREATE POLICY tenant_isolation ON rooms
  USING (tenant_id = current_setting('app.current_tenant')::uuid);
```

## 🔑 关键决策

### 决策 1: CRDT - 自研 vs 集成

**推荐**: ✅ **集成 yrs**

**理由**:
- 成熟稳定、社区活跃
- 节省 6-9 周开发时间
- 降低技术风险

---

### 决策 2: SFU - 自研 vs 集成

**推荐**: ✅ **集成 LiveKit**

**理由**:
- WebRTC 复杂度极高
- LiveKit 功能完整、性能优秀
- 节省 8-11 周开发时间

---

### 决策 3: 网关状态存储

**推荐**: ✅ **Phase 0 用 Redis，长期考虑 TiKV**

**理由**:
- Redis 简单成熟、社区支持好
- 长期可考虑分布式方案
